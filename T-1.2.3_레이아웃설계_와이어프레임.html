  17. G1 도어·서랍 = 앞판 있음 → 깊이에서 앞판 두께 차감 + 앞판 상부 찬넬 대상. 오픈 = 앞판 없음 → 깊이 차감·찬넬 없음, 트임 가능.

뷰 탭(정면/측면) · 트임 (v14 신규)

  1. V 뷰 탭(정면 / 측면): 캔버스 상단에서 보는 방향을 전환. 정면 = 칸 배치(기존 캔버스, 케이스 1~8). 측면(깊이) = 깊이 방향 단면(케이스 9). 정면도만으로 안 보이는 깊이·트임·로봇청소기 바닥 띄움·콘센트 보정을 측면에서 확인. 1차는 정면+측면 2종(평면도·3D는 추후).
  2. V2 측면 좌/우 토글: 측면은 별도 탭을 늘리지 않고 단일 탭 안에서 [좌측에서 ↔ 우측에서] 토글로 보는 쪽을 전환. 냉장고장 측면은 보통 좌·우 대칭이라 기본 한쪽만 보고, 트임처럼 좌우가 다른 경우만 토글로 반대쪽 확인(방안 1). 탭 줄은 정면/측면 2개로 유지.
  3. T1 트임(명세 반영, v31 위치 변경): 트임은 슬롯이 아니라 칸(장) 단위라 칸 패널(케이스 3)에 [바깥 면 트임] 토글로 둔다(이전 케이스 5에서 이동). 그 장이 가장 바깥쪽(맨 왼/오른) 열일 때만 노출. 명세의 "오픈장은 측면 한쪽을 트는 것"을, 사용자가 좌/우를 고르는 게 아니라 장 위치로 방향 자동(맨 왼=좌·맨 오른=우)으로 구체화.
  4. T2 가운데 열(양옆에 다른 장)이면 트임 토글 미노출. 오픈 슬롯이 없는 칸에서 트임을 켜면 "오픈 슬롯 + 트임으로 진행합니다" 안내를 띄우고 그 칸을 오픈으로 전환(새 칸 기본은 도어 1개라 도어인 경우가 많음). 트인 면은 측면 뷰에서 주황 칸으로 표시(케이스 9 W3).
  5. W1 측면 뷰 = 칸을 위→아래로 쌓고(높이), 가로로 깊이(앞=왼쪽·뒤=벽). 도어·서랍 슬롯은 앞쪽에 앞판(파란 띠) + 그 상부에 찬넬(주황 밴드)이 붙고, 앞판 두께만큼 몸통 깊이가 차감됨. (예: 600 → 578)
  6. W2 로봇청소기 칸: 장(몸통) 높이는 그대로 — 카르카스에 줄이는 선 없음(바닥판만 없음). 도어 앞판만 바닥에서 150mm 짧게 그려 띄움을 표현(도어 하단에만 점선·틈, 몸통엔 가로선 긋지 않음). 콘센트 보정(뒤 30mm)은 몸통 우측 끝 빗금.
  7. W3 트임 = 오픈 슬롯의 바깥 면 개방. 측면 뷰에서 주황 칸 + "↔ 트임(좌/우)" 뱃지 + 트인 쪽 화살표로 강조. 오픈 슬롯은 앞판이 없어 깊이 차감·찬넬 없음.
  8. W4 측면 뷰는 조회·확인용(깊이·앞판·찬넬·트임·바닥 띄움). 1차에서 측면 뷰 직접 편집 범위는 확정 필요(명세 동기화). 좌/우 토글(V2)로 트임처럼 좌우 다른 경우 반대쪽 확인.

슬롯 모델 · 속서랍 · 찬넬 (v16 재설계)

  1. S0 용어 고정: 열(column) = 세로 기둥(냉장고/장). 칸(cell) = 장 하나(한 열을 위·아래로 나눈 단위). 슬롯(slot) = 칸 안의 도어/서랍/오픈. 즉 칸이 슬롯을 담는 그릇. 한 칸의 장은 도어 슬롯·서랍 슬롯·오픈 슬롯을 위→아래로 조합(예: 도어 1 + 서랍 2단).
  2. S1 슬롯 종류 3가지: 도어(여닫는 문) / 서랍(전면 서랍 영역, 단 수 N) / 오픈(앞 트임). v15의 "오픈형/선반/서랍/속서랍 택1"을 폐기하고, 앞쪽 구성을 슬롯 종류 조합으로 표현(도어만/서랍만/도어+서랍/오픈이 전부 조합으로 나옴).
  3. S2 내부 채움: 도어·오픈 슬롯 내부를 일반 선반/레일 선반/속서랍 조합으로 채움. 속서랍은 도어 슬롯에서만(오픈은 가릴 도어가 없음). 서랍 슬롯은 서랍 자체라 내부 채움 없음.
  4. CH1 찬넬(앞판 상부) = 슬롯 단위(v27 변경): 도어·서랍 앞판 상부에 들어가는 가로 부재(손잡이 홈 + 옆 냉장고 도어 높이 라인 맞춤). 적용·높이 모두 슬롯에서 — 도어 슬롯 = 슬롯 편집 시트(케이스 5)에서 [이 도어 앞판에 찬넬 적용] 토글 + 높이를 개별 입력(같은 칸 안 도어끼리 달라도 됨, 라인 맞춤 필요 시) / 서랍 슬롯 = 각 단 앞판 상부에 항상 들어가며(끌 수 없음) 높이만 슬롯에서 설정(v32) / 오픈 슬롯 = 앞판 없어 제외. (이전 v16~v26의 '칸 단위 1개' 방식에서 변경.) 캔버스엔 주황 밴드(서랍은 단마다).
  5. CH3 상부 가림막(현장 호칭도 '찬넬', 별개 부재): 위에서 보면 ㅁ자인 가로 부재로, 장과 장 사이(또는 장 위)에 들어가 옆 냉장고와 높이를 맞춘다. 입력 = 장 칸 패널(케이스 3)의 [상부 가림막] 토글(마커 TG), 켜면 높이는 냉장고와의 높이 차이만큼 자동 계산(별도 입력 없음). 앞판 상부 '찬넬'과는 다른 부재라 명세에서도 명칭 분리.
  6. TG 상부 가림막 토글(v24): 장 칸 패널에 노출. 옆 냉장고와 높이를 맞추는 실제 채움 부재(높이는 냉장고 높이 차이만큼 자동). 도어 높이 정렬(라인 맞춤)은 프리셋으로 따로 적용 — 상부 가림막은 그와 별개의 물리 부재.
  7. AB 앞밴드 유무 토글(v24): 냉장고 칸 패널(케이스 4)에 노출. 냉장고장 앞면 가림막(세로·가로). 1차엔 유무만 — 켜면 작업지시서에 "있음" 노출, 상세 재단·치수는 추후.
  8. S3 깊이 차감·바닥: 도어/서랍 앞판이 있으면 그 깊이만큼 차감(측면 뷰에서 확인). 바닥과 면한 칸은 걸레받이(기본·도어 50mm 차감) ↔ 로봇청소기(바닥판 없음·150mm 띄움) 택일, 칸 단위.

마감 설정 — 색상 콤보박스 (케이스 8 · 케이스 3 패널 · v13)

  1. M 칸 액션 패널 "마감" 행: 선택한 칸의 도어 색상을 보여준다. 기본은 "전체 따름"(점선 콤보박스) — 누르면 케이스 8 시트가 열려 그 칸만 개별 색으로 덮어쓸 수 있다. 개별 지정된 칸이면 [전체로 되돌리기]로 다시 전체 기본값을 따르게 한다.
  2. N 마감 설정 시트: 색상은 콤보박스(입력창에 타이핑하면 목록이 실시간 필터링 + 선택)로 받는다. 목록에 있는 색만 선택 가능(자유 입력 불가). 도어 색상·홈바 색상은 같은 색상 목록을 공유한다. 색상 목록 값은 추후 운영에서 제공(코드 상수). 진입은 두 곳: 칸 미선택 시 패널의 [🎨 마감 설정] 카드(전체 도어 색상, 마커 O) + 칸 선택 후 패널 "마감" 행 콤보(그 칸만 override, 마커 M). 오픈장 칸이면 홈바 색상 항목도 함께.
  3. N1 도어 색상 — 전체 기본값: 마감 설정에서 한 번 지정하면 모든 장 도어의 기본 색이 된다.
  4. N2 전체 기본값을 바꾸면 "전체 따름" 칸만 따라 바뀌고, 칸에서 개별 지정한 색은 유지된다(칸 override 우선).
  5. N3 홈바 색상 — 오픈장 칸에서 직접 지정: 별도 전체 기본값을 두지 않고, 오픈장 칸을 선택했을 때만 이 칸의 홈바 색상을 시트에서 지정한다(도어 색상과 같은 콤보박스·같은 목록).
  6. N4 오픈장이 아닌 칸에는 홈바 색상 항목이 표시되지 않는다. (명세 T-1.2.3 "마감 설정" 반영 완료 — 색상 목록 값만 추후 제공.)

냉장고 모델 시트 — 브랜드 탭 (케이스 4-2 · v37 신규)

  1. E 모델 시트 상단에 브랜드 탭(LG / 삼성 / 기타)을 두고, 탭을 누르면 해당 브랜드 모델만 목록에 표시(다른 브랜드는 숨김). 기본 선택 = LG. [직접 입력]은 브랜드와 무관하게 항상 노출(탭 아래 고정). 명세의 "브랜드(삼성/LG) 셀렉트"를 셀렉트 박스가 아니라 상단 탭 필터로 구체화 — 명세 동기화 필요. 현재 기타 탭은 등록 모델이 없어 빈 상태 안내만 표시(직접 입력 유도).

장 프리셋 — 게이트(등분) 방식 (케이스 2-2 · v7)

  1. P1 진입: 케이스 2(요소 선택)에서 [장]을 누르면 이 시트. 상단 게이트(1·2·3·4등분·자유구성)로 칸 수를 먼저 고르면 아래에 그 등분의 변형 카드만 표시 → 스크롤 없이 비교. 카드 1장 = 장 1열 통째 생성. 도어 명칭은 도어 있는 칸만 위→아래 D1·D2·D3.
  2. P2 1등분 → 1도어(선반 2단) 1장. 유일하게 칸 속까지 정해진 프리셋(나머지는 분할만, 슬롯·속은 적용 후 [슬롯 편집]에서).
  3. P3 2등분 → 균등 / 상부 맞춤(D1=상부장 라인) / 하부 맞춤(D2=냉장고 하단 라인) 3장.
  4. P4 4등분 → 변형 추후 직접 채움(예약). 게이트 칸은 두되 변형 카드는 비워둠.
  5. P5 3등분 → 균등 / 상하 맞춤(D1=상부장·D3=냉장고 하단) / 홈바형 3장.
  6. P6 홈바형 → 칸은 3등분인데 가운데 도어 없이 오픈. 도어 개수가 아니라 등분(칸 수)으로 묶이므로 3등분에 속함. 도어는 위·아래 D1·D2.
  7. P7 자유구성 → 게이트에서 누르면 변형 카드 없이 바로 빈 장 1열 삽입(기존 동작).
  8. P8 정렬 기준선(파란 점선): 냉장고 모델 데이터(fridgeData)의 도어 분할 높이를 사용. 냉장고 없음·분할 정보 없는 모델이면 균등 등 대략 분할로 폴백 — 적용 후 칸 추가·속·크기 조정 가능(프리셋은 출발점).

코너(ㄴ) 레이아웃 — 면 전환 방식 (케이스 C1~C2 · v38)

  1. 10 코너(ㄴ)일 때만 캔버스 위에 면 전환 탭이 생긴다 — [면 A · 가로 벽] [면 B · 세로 벽]. 일자는 이 탭 없이 단일 캔버스. 한 번에 한 면을 작업하고(모바일에서 두 면 동시 표시는 각 면이 작아져 편집이 불편), 두 면 관계는 코너 미니맵으로 보조.
  2. 11 코너 미니맵: 위에서 본 ㄴ자 평면. 지금 보고 있는 면을 파랑으로 강조. 주 면(A)이 코너를 끝까지 차지하고 종속 면(B)을 바깥에서 감싸는 관계를 보여줌.
  3. 12 면 A는 주 면이라 면판 길이 = 벽 A 길이 + 면 B 깊이(2단계에서 정한 주 면 지정·두 벽 길이가 넘어옴). 캔버스 전체 폭은 이 면판 길이. 오른쪽 점선이 코너(면 B와 만나는 경계).
  4. 13 양면 허용: 면 A·면 B 모두 냉장고도 장도 놓을 수 있고, 각 면 안에서 독립적으로 열·칸·슬롯을 짠다. 높이(전체·서라운드전)는 두 면 공통(2단계 입력)이라 면을 바꿔도 세로 스케일이 같다.
  5. 14 면 B 탭으로 바꾸면 면 B 캔버스 + 미니맵 면 B 강조로 전환. 한 면 안의 편집 UI(열 추가·칸 나누기·슬롯 편집·측면 뷰·마감·모델 변경 등)는 케이스 1~9와 100% 동일.
  6. 15 면 B는 종속 면이라 면판 길이 = 벽 B 길이 그대로(가산 없음). 코너 쪽 끝은 면 A가 바깥에서 덮으므로 그만큼 물러나지 않고 그대로 끝난다.
  7. 면판 길이 규칙(감쌈 ㄴ): 주 면 = 벽 길이 + 종속 면 깊이 / 종속 면 = 벽 길이 그대로. 측정 벽 길이는 그대로 두고 재단 면판만 가산(각도 90° 고정). 시스템이 자동 계산하므로 기사가 확인·조작할 화면은 두지 않는다(이전 C3 화면 제거). 안쪽 ㄱ자는 1차 제외 — 싱크대 작업 때 데드 스페이스 처리와 함께.
  8. 완료 시 작업지시서: 코너 품목은 면별 정면도 2장(면 A·면 B) + 평면도 1장(ㄴ자 배치·주 면/종속 면·맞댐 관계), 본체 합판 재단도는 두 면 품목 합산 산출. 이는 레이아웃 편집 화면이 아니라 작업지시서(A-2.3) 명세 사항.
