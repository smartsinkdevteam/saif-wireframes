명세 반영 사항 (기능 명세서 T-1.1 기준)

  1. 1 상단 헤더: 화면명 "작업 목록" + 본인 이름. 진입: 로그인 직후 / 하단 탭 바 [작업 목록]. 본인 배정 주문만 표시.
  2. 2 검색: 고객명·주소 부분 일치, 2자 이상 입력 시 적용(관리자 주문 목록과 동일 방식).
  3. 3 작업 유형 필터 — 담당 업무별: 실측만 = 전체/실측, 시공만 = 전체/시공, 겸임 = 전체/실측/시공. 케이스 1은 겸임 기사 기준. + 결제 상태 필터(계약금 대기/착수금 대기/잔금 대기/잔금 완료/환불 멀티 셀렉트, 미선택 시 전체). ※ 06-12 결정: 진행 상태 필터 제거 — 기사에게는 입금(결제) 상태 확인이 중요.
  4. 4 날짜 필터(캘린더 — 하루 이동, 명세 반영): 캘린더에서 하루씩 이동(기본값 오늘). 이전/다음 날짜 화살표(‹ ›) + 날짜 선택(가운데 날짜 탭 시 달력 피커). "오늘/이번 주/전체" 같은 기간 칩이 아니라 단일 날짜 네비게이터. 필터 변경 시 리스트 즉시 갱신.
  5. 5 작업 카드: 작업 유형 배지(실측·시공 색 구분) + 고객명 + 주소 요약 + 결제 상태 배지(06-12 결정, 진행 상태 배지 대체) + 배정 일정 날짜(YYYY-MM-DD) + 완료 여부 체크. 카드 클릭 → [T-1.1.1 주문 상세 (기사)].
  6. 6 필터 결과 0건: "조건에 맞는 작업이 없습니다."
  7. 7 배정 없음: "배정된 작업이 없습니다." (1차에는 푸시 알림 없음 — 기사가 직접 목록을 열어 확인.)
  8. 8 로딩 실패: "작업 목록을 불러올 수 없습니다. 화면을 아래로 당겨 새로고침해주세요." — 명세 문구 그대로.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 날짜 네비게이터의 형태(‹ 날짜 › + 요일·"오늘" 배지, 가운데 탭 시 달력)는 명세의 "하루씩 이동 + 이전/다음 화살표 + 날짜 선택"을 시각화한 것 — 세부 UI는 디자인 단계 확정.
  2. 작업 유형·결제 상태를 칩 UI로 표현 — UI 형식은 명세에 정의 없음.
  3. 실측=파랑, 시공=청록 배지 색 매핑은 임의 지정("시각 구분"만 명세에 정의). 완료 표시(✓ 완료)의 위치·형태도 임의.
  4. 당겨서 새로고침 화살표(↓) 표현은 임의. 샘플 데이터는 전부 가상의 예시.
