명세 반영 사항 (기능 명세서 A-3.2 기준)

  1. 1 진입: [A-3.1 기사 목록]의 [+ 새 기사 등록]. 완료 후 [A-3.1]로 복귀.
  2. 2 휴대전화번호 = 로그인 아이디. 11자리 휴대폰, 자동 하이픈(010-XXXX-XXXX), 전체 시스템 중복 불가. 1차에는 기사 본인 변경 불가(관리자는 A-3.4 기사 수정에서 변경 가능).
  3. 3 초기 비밀번호: 8자 이상·2종류 이상 조합. [자동 생성] 클릭 시 서버 규칙에 맞는 랜덤 비밀번호 생성 → 필드 채움 + 표시.
  4. 4 담당 업무: 실측/시공 체크박스, 1개 이상. 둘 다 체크 = 겸임 기사.
  5. 5 취급 가능 품목: 다품목 반영 — 냉장고장·붙박이장·신발장·싱크대 체크박스 그룹, 1개 이상 선택. 주문 배정 시 후보 필터에 사용. (수정 A-3.4와 동일 구성)
  6. 6 선택 항목 4종: 차종(50자) / 차량번호(20자, 한글·숫자) / 집주소(카카오 주소 검색 + 상세 100자) / 사업자등록증 이미지.
  7. 7 사업자등록증: JPG/PNG/HEIC/PDF, 최대 10MB, 1장. 정산 증빙·세금계산서 발행 등에 활용. A-3.3에서 조회·교체 가능.
  8. 8 [저장]: 필수 필드 유효 시 계정 생성 → [A-3.1]로 복귀, 완료 토스트.
  9. 9 휴대전화번호 중복: "이미 등록된 휴대전화번호입니다."
  10. A 파일 에러: 용량 초과 "10MB 이하의 파일만 업로드할 수 있습니다." / 형식 오류 "JPG/PNG/HEIC/PDF 형식만 업로드할 수 있습니다."
  11. B [취소]: 입력 내용 있으면 확인 모달 후 복귀.
  12. C 완료 토스트 후 [A-3.1] 복귀.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 취급 가능 품목 4종(냉장고장·붙박이장·신발장·싱크대)은 다품목 반영. 품목 종류 목록 확정·회사별 가변 여부는 명세 동기화 필요(등록·수정·기사 목록 요약 동일 기준).
  2. 섹션 2분할("계정 정보(필수)" / "추가 정보(선택)")과 섹션명, 2단 그리드 배치는 명세에 그룹 정의가 없어 임의 구성 — 확정 필요.
  3. 완료 토스트 문구, 취소 모달 문구·버튼 라벨, 비밀번호 자동 생성 예시값은 임의 작성.
  4. 에러 메시지 표시 위치(각 필드 하단)는 명세 A-3.2에 명시가 없어 A-2.2 패턴을 따름.
  5. 샘플 데이터는 전부 가상의 예시.
