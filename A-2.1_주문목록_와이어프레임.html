명세 반영 사항 (기능 명세서 A-2.1 기준)

  1. 1 진입 경로: 좌측 사이드바 "주문 관리" — IA v0.5 1depth 메뉴 트리(대시보드 / 주문 관리 / 기사 관리 / 정산(추후)) 기준. 화면명은 명세대로 "주문 목록" 유지(메뉴명 ≠ 화면명).
  2. 2 [+ 새 주문 등록] 클릭 시 A-2.2 주문 등록 화면으로 이동. 항상 활성.
  3. 3 검색 대상: 고객명·주소·핸드폰 번호·특이사항 부분 일치. 2자 이상 입력 시 적용, 디바운스 300ms.
  4. 4 진행 상태(접수/실측 완료/생산/시공 완료/종료/보류/취소)·결제 상태(계약금 대기/착수금 대기/잔금 대기/잔금 완료/환불) 멀티 셀렉트. 미선택 시 전체.
  5. 5 행 클릭 시 해당 A-2.3 주문 상세로 이동. 정렬 가능 컬럼은 실측일·등록일·수정일 3개뿐이며 등록일이 기본 내림차순(↓). 헤더 클릭 시 오름차순↔내림차순 전환.
  6. 6 페이지네이션: 페이지당 50건 기본.
  7. 7 보조 네비게이션: 우측 상단 프로필 드롭다운(내 정보, 비밀번호 변경, 로그아웃) — IA "사용자별 네비게이션 패턴" 기준.
  8. 8 시공범위 컬럼(다품목 요약 반영): 품목 요약 표기. 1종이면 품목명 그대로("냉장고장"), 여러 종류면 "대표품목 외 N건"(예: "냉장고장 외 2건" = 냉장고장+2종). 한 주문에 여러 종류 품목을 받을 수 있음(같은 종류는 1개씩). 대표품목 선정 기준(첫 품목/우선순위)은 개발 단계 확정.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 샘플 데이터(고객명·주소·날짜·건수 128건, 시공범위 "냉장고장 외 2건" 등)는 전부 가상의 예시. "외 N건"의 N·대표품목 표기 형식은 명세 예시("냉장고장 외 2건") 기준이되 최종 포맷은 확정 필요.
  2. 상태 배지의 색상 매핑(접수=회색, 실측 완료=파랑, 생산=주황 등)은 문서에 정의가 없어 임의 지정. 디자인 단계에서 확정 필요.
  3. "새 주문 등록" 버튼의 강조(검정 배경) 스타일, 사이드바 메뉴의 이모지 아이콘은 임의 표현.
  4. 추후 범위 메뉴(정산)는 와이어프레임에서 제외 — 1차 오픈 메뉴(대시보드·주문 관리·기사 관리)만 노출.
  5. 케이스 2~4의 빈 상태/에러 문구·버튼은 명세의 "빈 상태"·"에러 메시지" 정의 그대로. 상태 영역의 아이콘(⌀·＋·⚠)과 중앙 배치 레이아웃은 임의 표현. 케이스 4의 [다시 시도] 버튼은 2026-06-12 결정 사항(명세 동기화 완료).
  6. 케이스 3(주문 없음)에서 검색·필터 영역을 노출 상태로 그렸는데, 데이터가 없을 때 숨길지는 명세에 정의 없음 — 확인 필요.
