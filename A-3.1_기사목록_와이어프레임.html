명세 반영 사항 (기능 명세서 A-3.1 기준)

  1. 1 진입 경로: 좌측 사이드바 "기사 관리" (IA v0.5 1depth). 화면명은 명세대로 "기사 목록" 유지(메뉴명 ≠ 화면명).
  2. 2 [+ 새 기사 등록] → [A-3.2 기사 등록] 이동.
  3. 3 검색·필터: 검색창(이름/연락처), 담당 업무 필터, 활성/비활성 필터.
  4. 4 담당 업무: 실측 / 시공 / 실측·시공 배지.
  5. 5 취급 가능 품목(다품목 요약 반영): 체크된 품목 요약 표기. 1종이면 품목명 그대로("냉장고장"), 여러 개면 "대표품목 외 N"(예: "냉장고장 외 3"). 기사 등록(A-3.2)에서 여러 품목 선택 가능.
  6. 6 활성/비활성 필터 기본값 "활성"(명세 반영): 기본 화면은 활성 기사만 표시. "전체"·"비활성"으로 전환 가능.
  7. 7 수정일 컬럼(명세 반영): 마지막 수정 YYYY-MM-DD. 컬럼 = 이름·연락처·담당 업무·취급 가능 품목·상태·등록일·수정일. 행 클릭 시 [A-3.3 기사 상세] 이동.
  8. 8 빈 상태: "등록된 기사가 없습니다." + [새 기사 등록] 버튼.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 취급 가능 품목 "외 N" 표기 형식·대표품목 선정 기준은 명세 예시("냉장고장 외 3") 기준이되 최종 포맷은 확정 필요.
  2. 담당 업무·상태 배지의 색상 매핑(실측=파랑, 시공=청록, 실측·시공=보라, 활성=초록, 비활성=회색)과 비활성 행 회색 처리는 임의 지정.
  3. 활성/비활성 필터의 라벨 표기("활성/비활성: 활성")는 임의 — 기본값 "활성"만 명세 정의.
  4. 검색 적용 조건(글자 수·디바운스), 정렬, 페이지네이션, 필터 결과 0건·로딩 실패 상태는 명세 A-3.1에 정의 없음 — 와이어프레임에 넣지 않음. A-2.1 수준 정의가 필요하면 명세 보강 후 반영.
  5. 샘플 데이터는 전부 가상의 예시.
