명세 반영 사항 (기능 명세서 A-3.3 기준)

  1. 1 헤더: 기사 이름 + 상태 배지 + [수정] + [임시 비밀번호 발급] + [활성/비활성] 토글. 진입은 [A-3.1]에서 행 클릭.
  2. 2 인적사항: 이름·휴대전화번호(아이디)·담당 업무·취급 품목·등록일 + 선택 항목(차종·차량번호·집주소·사업자등록증). 선택 항목 미등록 시 "미등록" 표시.
  3. 3 사업자등록증 이미지: 다운로드·교체 가능.
  4. 4 [수정]: 편집 가능한 필드를 인라인 전환. 수정 가능: 이름·휴대전화번호·담당 업무·취급 품목·차종·차량번호·집주소·사업자등록증.
  5. 5 휴대전화번호 변경: 전체 시스템 중복 검증(중복 시 "이미 등록된 휴대전화번호입니다."), 변경 후 새 번호로 로그인 필요.
  6. 6 임시 비밀번호 발급 확인 모달: "OOO 기사의 임시 비밀번호를 발급하시겠습니까? 기존 비밀번호는 즉시 무효화됩니다." — 명세 문구 그대로.
  7. 7 결과 모달: 임시 비밀번호 표시 + [복사]. 다음 로그인 시 강제 변경(S-0.4 자동 열림). 같은 화면 세션에서는 재열람 가능하나 새로고침 시 재조회 불가.
  8. 8 비활성 토글: 확인 모달 후 변경. 비활성 시 로그인 차단. 다가올 배정 일정이 있으면 경고 "해당 기사가 배정된 다가올 일정이 X건 있습니다. 재배정이 필요합니다." — 명세 문구 그대로.
  9. 9 담당 업무 축소 에러: "현재 배정된 일정이 있어 담당 업무를 축소할 수 없습니다. 일정을 먼저 재배정해주세요." / 발급 실패: "임시 비밀번호 발급에 실패했습니다..."

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 스케줄 섹션(이번 달 배정 일정 리스트)은 06-12 결정으로 화면에서 제외 — 명세 동기화 완료. 비활성 토글 경고의 일정 건수 집계는 그대로 유지.
  2. 기사 삭제 버튼은 두지 않음 — 명세 비고(활성/비활성 토글로만 관리) 반영.
  3. 수정 모드의 헤더 버튼([취소]·[저장]) 구성, 비활성 확인 모달의 본문 문구·버튼 라벨, 임시 비밀번호 결과 모달의 안내 문구·예시값은 명세에 상세 정의 없음 — 임의 작성, 확정 필요.
  4. 토글 UI 형태는 임의 표현. 샘플 데이터는 전부 가상의 예시.
