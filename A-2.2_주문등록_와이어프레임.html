명세 반영 사항 (기능 명세서 A-2.2 기준)

  1. 1 진입: [A-2.1 주문 목록]의 [+ 새 주문 등록]. 저장 완료 후 생성된 [A-2.3 주문 상세]로 이동.
  2. 2 연락처: 숫자·하이픈만, 입력 중 자동 하이픈. 11자리 휴대전화 또는 9–10자리 지역번호 허용.
  3. 3 주소: 카카오 우편번호 서비스 팝업 → 자동 채움(읽기 전용). 검색 DB 미등재 주소는 [직접 입력] 폴백.
  4. 4 시공 품목(다품목·인스턴스): 품목 줄 목록. 한 줄 = order_items 인스턴스 1개. [＋ 품목 추가]로 줄을 늘리고 줄마다 종류(드롭다운)·위치명 라벨(선택)을 입력. 같은 종류도 줄을 추가해 여러 개 등록 가능(예: 냉장고장 2개). 위치명을 비우면 자동 번호("냉장고장 1·2")가 유지되고, 비운 줄은 실측 기사가 현장에서 품목 탭으로 채울 수 있음. 최소 1개. 종전 체크박스 그리드를 폐기 — 줄을 추가하는 것이 곧 품목 추가이고, 시공범위는 order_items에서 도출(단일 출처). 종류 옵션(냉장고장·붙박이장·신발장·싱크대)은 운영 정의·공통 코드.
  5. 5 주문 경로·상담 경로(신규 반영): 주문 경로(선택, 셀렉트 — 박람회/블로그/인스타그램/지인 소개/숨고/기타), 상담 경로(선택, 셀렉트 — 톡톡/카톡/전화/숨고채팅). 둘 다 공통 코드(보기값).
  6. 6 실측일: 오늘 이후~최대 6개월. 필드 클릭 시 [전체 스케줄] 모달(케이스 3).
  7. 7 계약금 있음: 기본 미체크. 체크 시 금액 필드 활성·필수 전환, 기본값 50,000원(수정 가능). 해제 시 비활성·값 초기화. 0 이상 정수, 최대 1억원.
  8. 8 결제방법: 평소 선택이나 "계약금 있음" 체크 시 필수. "카드" 선택 시 카드사·끝 4자리 필수(케이스 2), "기타" 선택 시 특이사항 필수(케이스 4).
  9. 9 고객 제공 사진: 분류 "고객 제공" 자동 설정, 주문 저장 시 함께 저장. 장당 10MB·분류별 30장·JPG/PNG/HEIC.
  10. 10 [저장]: 필수 필드 유효 시. 진행 상태=접수, 결제 상태=계약금 있음이면 "계약금 대기"/아니면 "착수금 대기"로 생성.
  11. A 케이스 2: 계약금 체크 직후 50,000원 자동 채움 상태.
  12. B 카드사: 고정 목록 드롭다운(신한·삼성·현대·KB국민 등 + 기타 직접입력 최대 20자). 1차는 코드 상수 고정.
  13. C [전체 스케줄] 모달: 월 캘린더에 날짜별 일정을 건별 스택으로 표시(작업 유형·배정 기사, 06-12 결정), 이전/다음 달 이동, 오늘 강조. 날짜 클릭 시 실측일 채움+닫힘. A-2.3 배정 모달에서 재사용.
  14. D [취소]: 입력 내용 있으면 확인 모달 후 [A-2.1]로 복귀.
  15. E 저장 실패(서버 오류) 토스트: "주문 저장에 실패했습니다. 잠시 후 다시 시도해주세요." — 명세의 폼 상단 토스트.
  16. F 저장 완료 토스트(06-12 결정): 저장 성공 시 폼 상단 표시 후 [A-2.3 주문 상세]로 이동.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 시공 품목 종류 4종(냉장고장·붙박이장·신발장·싱크대)·주문/상담 경로 옵션 값은 명세 표기. 1차 노출 품목·경로 옵션의 최종 목록은 운영 정의·공통 코드 상세(개발 단계). 품목 줄 목록의 행 레이아웃(종류 드롭다운+위치명+삭제)·[＋ 품목 추가] 위치·위치명 자동 번호 표기는 임의 — 확정 필요.
  2. 섹션 4분할(고객 정보/현장·계약/특이사항/사진), 2단 그리드 배치·섹션 카드 스타일·필드 순서는 임의.
  3. 필수 표시(*), 플레이스홀더 문구, 토스트의 아이콘·색상은 임의 표현.
  4. 케이스 3 캘린더 스택 칩의 표기 형식("실측 김기사")과 [닫기] 버튼, 케이스 5 모달의 문구·버튼 라벨은 명세에 상세 정의 없음 — 임의 작성, 확정 필요.
  5. 케이스 4는 대표 에러만 표시(카드사·카드 끝 4자리·계약금 음수 에러 등은 명세 표 참고).
  6. 샘플 데이터는 전부 가상의 예시.
