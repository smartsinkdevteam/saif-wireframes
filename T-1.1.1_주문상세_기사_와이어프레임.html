T-1.1.1 주문 상세 (기사) 모바일 웹·PWA

위 컨트롤로 역할(실측/시공)과 진행 단계를 바꾸면 화면이 그에 맞게 갱신됩니다. 품목 탭, 하단 완료 버튼, 입금 확인·품목 변경 시트, 실측 조회 화면은 모두 클릭해 동작을 확인할 수 있어요.

명세 반영 사항 (기능 명세서 T-1.1.1 기준)

  1. 진입: [T-1.1 작업 목록] 카드 클릭 → 주문 상세. 본인 담당 업무·진행 상태에 따라 구성·완료 액션이 달라짐(역할 토글로 확인).
  2. 고객/현장 정보: 고객명·주소·연락처·특이사항(읽기 전용) + 고객 제공 사진(조회만). 주문 전체 단위.
  3. 일정·결제 + 입금 확인: 결제 상태는 계약금 대기 → 착수금 대기 → 잔금 대기 → 잔금 완료(+환불)로 진행 상태와 독립(데이터 모델 2축). 계약금은 주문 등록 시 거는 고객만 — 안 걸면 계약금 대기로 시작. 단 실측 기사 주문 상세에 오는 주문은 계약금 대기 또는 착수금 대기 상태이고, 계약금을 걸었든 안 걸었든 기사가 계약 성공 시 착수금을 받아오므로 화면 표시는 '착수금 대기'로 통일(잔금 대기·완료는 이 화면에 나타나지 않음). 착수금은 계약서 작성(총액 확정) 시 수령하므로 [착수금 입금 확인]은 실측 완료(계약 확정) 후 노출되고, 누르면 결제 상태가 잔금 대기로 전이(이후 잔금 입금 확인은 관리자). 금액 — 총액 = 계약서 행 합계, 착수금 = 총액 × 30%, 잔금 = 총액 − 계약금 − 착수금. 입금 확인 권한: 착수금 = 실측 기사·관리자 / 잔금 = 관리자만(계좌이체 포함). 시공 기사 화면에는 일정·결제 섹션 자체를 노출하지 않음.
  4. 품목 탭 위계: 실측 상세(치수·메모·사진)와 레이아웃 미리보기는 품목 종속이라 탭 콘텐츠 안에 묶여 탭 전환 시 통째로 교체. 계약 정보(합계)는 주문 전체 단위라 탭 바깥 상위에 고정.
  5. 실측 시작 / 비냉장고장: 냉장고장만 [실측 시작]→위저드(T-1.2.1). 비냉장고장(붙박이장·신발장·싱크대)은 위저드 없이 [실측 완료] 버튼으로 측정완료 처리(측정 데이터·작업지시서 없음, 계약 금액 라인만).
  6. 실측 완료(주문 단위) 활성 조건: 모든 품목 측정완료 + 계약서 확정 + 실측 참고 사진 1장 이상. 미충족 시 부족 항목 안내.
  7. 계약서 작성 → 확정 → 실측 완료: 전 품목 측정완료 시 [계약서 작성](T-1.3) 활성. 계약 확정 전까지 [실측 완료] 비활성("계약서 확정 필요"). 계약 금액은 계약서 작성 화면에서 항목별 input으로 입력(합계 자동 합산)하므로, 작성 전 단계에는 금액·합계를 표시하지 않고 [계약서 작성] 버튼만 노출. 확정된 금액은 실측 완료 단계에서 표시.
  8. 작업지시서 게이팅: 작업지시서는 계약 확정 + 진행 상태 실측 완료 이후 발행되는 산출물이므로 그 이전 단계에는 노출되지 않음. 실측 완료 후 주문 상세에서 인쇄(window.print(), 관리자와 동일한 방식 · PDF 파일 저장 없음).
  9. 시공 기사: 일정·결제 미노출, [실측 조회](읽기 전용 전체 페이지), [품목 변경](현장 품목 추가 시트 — 본인 실측 담당 품목만, 취소는 실측 기사). 시공은 품목별 [시공 시작]→위저드(T-1.4).
  10. 시공 완료(주문 단위) 활성 조건: 모든 품목 시공완료(시공 정보 저장 + 완료 사진 1장 이상). 이후 보수는 AS(추후).
  11. 에러 토스트: 사진 용량/장수/업로드 실패/완료 처리 실패 — 명세 문구 그대로(컨트롤 바의 "에러 토스트 예시 보기").

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 동적 전환은 검토용: 역할·단계 전환과 버튼 동작은 위계·흐름 확인을 위한 것이며, 실제 화면 전환·애니메이션·상태 머신은 구현 시 확정. 완료 버튼·수정 버튼 등은 실제 위저드/계약 화면(별도 명세) 대신 안내 토스트로 표현.
  2. 레이아웃 미리보기 노출: 실측 기사 주문 상세의 도면 미리보기는 기능 명세서에 명시 없음. 시공 기사 [실측 조회] 도식을 품목 탭 안에 축소 재사용한 것 — 노출 여부·렌더 방식 확정 필요.
  3. 작업지시서 노출 단위: "주문 단위 실측 완료 이후 노출"로 잡았으나, "품목별 냉장고장 측정완료 시 바로 노출"인지 명세 확인 필요.
  4. 입금 확인 시트의 금액·결제방법 UI, 예상 잔금 자동 계산, 카드사 셀렉트는 명세 상세 정의 없음. [품목 변경]은 바텀시트, [실측 조회]는 전체 페이지로 표현(형태 정의 없음).
  5. 한 주문에 여러 종류 품목(같은 종류 1개씩)은 1차 동작에 해당. 같은 종류 여러 개·별칭은 추후(데이터 모델만 선반영).
  6. 섹션 구획·버튼 스타일·색상·샘플 데이터(치수·금액·연락처 등)는 전부 임의/가상.
  7. [품목 변경] 시트는 시공 기사가 본인 실측 담당 품목만 추가할 수 있고 취소는 불가(취소는 실측 기사). 명세·PRD·유저 플로우와 정합.
