명세 반영 사항 (기능 명세서 S-0.4 기준)

  1. 1 새 비밀번호: 필수, 최소 8자·최대 100자, 영문·숫자·특수문자 중 2종류 이상 조합, 현재 비밀번호와 동일 불가.
  2. 2 입력 중 실시간 복잡도 가이드 표시 — 명세 그대로(체크 형태 표현은 임의).
  3. 3 [취소]: 일반 흐름에서는 항상 활성 → 모달 닫기.
  4. 4 [저장]: 3개 필드 모두 입력 시 활성 → 서버 검증 → 성공 시 "비밀번호가 변경되었습니다." 표시 후 모달 닫힘.
  5. 5 진입 경로 3종: 관리자 프로필 드롭다운 → [비밀번호 변경] / 기사 설정 탭([T-2.1]) → [비밀번호 변경] / 임시 비밀번호 로그인 시 자동 열림(강제 변경). 관리자가 [A-3.3]에서 기사 비밀번호를 재설정한 경우에도 해당 기사 다음 로그인 시 자동 열림.
  6. 6 강제 변경 안내 배너 — 흐름은 명세 그대로, 문구는 임의.
  7. 7 강제 변경 흐름에서는 이미 임시 비밀번호로 인증을 마친 상태이므로 "현재 비밀번호" 필드를 숨기고 검증 생략 — 명세 그대로.
  8. 8 강제 변경 흐름에서 [취소] 비활성(변경 완료 전에는 닫을 수 없음). 변경 완료 시 원래 이동하려던 화면(관리자=[A-1.1 대시보드], 기사=[T-1.1 작업 목록])으로 이동 — 명세 그대로.
  9. 9 "현재 비밀번호가 올바르지 않습니다." — 명세 문구 그대로.
  10. A "영문·숫자·특수문자 중 2종류 이상을 조합해 8자 이상으로 입력해주세요." / "현재 비밀번호와 다른 비밀번호를 입력해주세요." — 명세 문구 그대로(두 에러는 상황별 1개씩 노출, 표현 모음).
  11. B "두 비밀번호가 일치하지 않습니다." — 명세 문구 그대로.
  12. C 성공 "비밀번호가 변경되었습니다." / 서버 오류 "일시적인 문제가 발생했습니다. 잠시 후 다시 시도해주세요." — 명세 문구 그대로.

⚠️ 문서에 없는 임의 표현

  1. 강제 변경 모달의 제목 "새 비밀번호 설정"과 경고 배너 문구는 임의 — S-0.3 안내문("새 비밀번호 설정 화면")과 톤을 맞춘 것, 확정 필요.
  2. 복잡도 가이드의 체크리스트(8자/영문/숫자/특수문자) 형태, placeholder, 눈 아이콘 토글 위치는 임의(눈 토글 자체는 S-0.1·S-0.2 패턴 준용).
  3. 케이스 3은 에러 문구 모음용으로 한 화면에 모두 표시 — 실제로는 상황별 해당 에러만 노출.
  4. 강제 변경 흐름 모달에 ✕ 닫기 아이콘 제거(닫기 불가 표현)는 임의 해석.
