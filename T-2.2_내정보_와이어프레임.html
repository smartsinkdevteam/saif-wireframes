계속 수정 나가기

명세 반영 사항 (기능 명세서 T-2.2 기준)

  1. 1 진입: [T-2.1 설정] → [내 정보]. 헤더: 화면명 "내 정보" + 좌측 [← 뒤로].
  2. 2 조회 전용 섹션: 이름·휴대전화번호 회색 처리 + 안내 문구(명세 그대로). 변경은 본인 인증 도입 후, 필요 시 관리자에게 요청.
  3. 3 편집 가능 섹션: 차종(50자)·차량번호(20자, 한글·숫자)·집주소(카카오 주소 검색 + 상세 100자)·사업자등록증. 모두 선택 항목.
  4. 4 사업자등록증: 기존 업로드본 미리보기 + [교체]. 새 파일 선택 시 기존 자동 대체. JPG/PNG/HEIC/PDF, 10MB, 1장.
  5. 5 [저장]: 변경 사항 있음 + 유효 시 → 본인 정보 업데이트, 완료 토스트, [T-2.1]로 복귀. 변경 즉시 [A-3.3 기사 상세]에도 반영.
  6. 6 변경 사항 없이 [저장]: "변경된 내용이 없습니다."
  7. 7 옵셔널 항목 미입력 허용, 미등록 시 "미등록" 표시.
  8. 8 파일 에러: 형식 "JPG/PNG/HEIC/PDF 형식만 업로드할 수 있습니다." / 용량 "10MB 이하의 파일만 업로드할 수 있습니다." / 저장 실패 "저장에 실패했습니다. 잠시 후 다시 시도해주세요."
  9. 9 [← 뒤로]: 변경 사항 있으면 확인 모달 표시 후 [T-2.1]로 복귀.

⚠️ 문서에 없는 임의 표현 (와이어프레임 작성자 판단)

  1. 뒤로가기 확인 모달의 문구·버튼 라벨, 저장 완료 토스트 문구는 명세에 정의 없음 — 임의 작성, 확정 필요.
  2. "미등록"을 입력 필드의 플레이스홀더 형태로 표현 — 표시 방식(텍스트만 vs 필드 내)은 명세에 정의 없음.
  3. 저장 버튼의 하단 고정 배치, 섹션 구분 스타일은 임의 표현. 샘플 데이터는 가상의 예시.
