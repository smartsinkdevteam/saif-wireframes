명세 반영 사항 (기능 명세서 A-2.4 기준)

  1. 1 진입: [A-2.3 주문 상세]의 [수정] 버튼. A-2.2와 동일한 폼 구성에 기존 값이 채워진 상태로 열림. 진행 상태 "종료"·"취소"는 A-2.3에서 [수정] 버튼 자체가 비활성이라 이 화면 진입 불가.
  2. 2 시공 품목(다품목·인스턴스): A-2.2와 동일한 품목 줄 목록(줄마다 종류 드롭다운 + 위치명, [＋ 품목 추가]). 접수 단계에서는 줄 추가·삭제·종류 변경 가능, 같은 종류도 여러 줄로 등록(각 줄 = order_items 인스턴스 1개), 위치명 비우면 자동 번호. 주문 경로·상담 경로도 A-2.2와 동일하게 포함.
  3. 3 고객 제공 사진: 기존 사진 조회·추가·삭제 가능 (A-2.3에서는 조회만, 추가·삭제는 이 화면에서).
  4. 4 [저장]: 필수 필드 유효 시. 주문 업데이트 + 수정일·수정자 자동 기록(A-2.1 목록의 수정일·작성자 컬럼 갱신) → [A-2.3]으로 복귀, 완료 토스트.
  5. 5 수정 제한: 진행 상태 "실측 완료" 이후 시공 품목 줄 목록 전체 잠금(줄 추가·삭제·종류 변경 불가) + "실측 완료 후 시공 품목은 수정할 수 없습니다." 등록된 품목 줄이 읽기 전용 회색으로 표시된다.
  6. 6 변경 사항 없이 [저장] 시도: "변경된 내용이 없습니다." 폼 상단 토스트.
  7. 7 저장 완료 토스트 후 [A-2.3 주문 상세] 복귀.

A-2.2와 동일하여 생략한 케이스

  1. 입력 필드 유효성 에러(고객 이름·연락처·주소·시공 품목·실측일·계약금·"기타" 결제 특이사항·"카드" 결제 카드사·끝 4자리) — A-2.2 케이스 4와 동일.
  2. 조건부 필드(계약금 체크·카드 선택) 동작 — A-2.2 케이스 2와 동일.
  3. [전체 스케줄] 모달 — A-2.2 케이스 3과 동일(건별 스택).
  4. [취소] 확인 모달(변경 사항 있을 때) — A-2.2 케이스 5와 동일, 복귀처만 [A-2.3]으로 다름.
