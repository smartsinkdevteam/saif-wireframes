명세 반영 사항 (기능 명세서 S-0.1 기준)

  1. 1 데스크톱 웹 접속 시 기본 화면. 세션 유효 시 이 화면을 건너뛰고 [A-1.1 대시보드]로 이동. 로그인 성공 시에도 [A-1.1]로 이동. 관리자는 자동 로그인 토큰 없이 세션 기반 유지(만료 시 재로그인).
  2. 2 비밀번호: 필수, 1자 이상 최대 100자, 기본 마스킹. 우측 눈 아이콘으로 표시/숨김 토글.
  3. 3 [로그인]: 이메일·비밀번호 둘 다 입력 시 활성 → 서버 인증 호출, 실패 시 에러 메시지.
  4. 4 [비밀번호를 잊으셨나요?] 링크: 항상 활성 → [S-0.3 비밀번호 재설정 안내]로 이동.
  5. 5 이메일: 필수, 이메일 형식(RFC 5322 준용), 최대 100자. 입력 시 우측 [X] 버튼으로 일괄 삭제.
  6. 6 두 필드 입력 완료 시 [로그인] 활성.
  7. 7 "이메일을 입력해주세요."(미입력 포커스 이탈) / "올바른 이메일 주소를 입력해주세요."(형식 오류) — 이메일 필드 하단, 명세 문구 그대로.
  8. 8 "비밀번호를 입력해주세요."(미입력 포커스 이탈) — 비밀번호 필드 하단, 명세 문구 그대로.
  9. 9 인증 실패: "이메일 또는 비밀번호가 올바르지 않습니다." — 폼 상단 토스트/배너. 이메일·비밀번호 중 어느 것이 틀렸는지 구분해 노출하지 않음(보안) — 명세 그대로.
  10. A "비활성화된 계정입니다. 관리자에게 문의해주세요." / "일시적인 문제가 발생했습니다. 잠시 후 다시 시도해주세요." — 폼 상단 토스트, 명세 문구 그대로.

⚠️ 문서에 없는 임의 표현

  1. 중앙 카드형 레이아웃, SAIF 로고·"관리자 로그인" 부제, 필드 placeholder 문구는 임의 — 확정 필요.
  2. 케이스 4에서 배너(인증 실패)와 토스트(비활성·서버 오류)를 한 화면에 함께 그린 것은 표현 모음용이며, 실제로는 상황별 1개씩 노출.
  3. 유효성 에러 시 [로그인] 비활성 처리(케이스 3)는 활성 조건(둘 다 "입력")의 해석 — 형식 오류 시에도 활성로 둘지 확정 필요.
